LiquidIoT device manager

The liquidIoT device manager manages information about IoT devices and apps deployed to them. Devices can register to the manager and also update their information later such as what applications have . been deployed to them. The manager can give a list of all devices and applications deployed on them. User can also query devices and applications with different criterions. Device manager manages also app api classes which define using swagger which kind of api an application has. The device manager can produce a swagger api description for an app deployed on a device based on the classes the app has.

put /apis/{className}
Create or replace the API description (classModify)
A new classs will be created if there already is not a class with the given name otherwise the existing class will be updated with the information in the request.

Path parameters

className (required)
Path Parameter — Name of the class

Request body

api (required)
Body Parameter — A swagger fragment that defines the api offered by apps that implement this class. A field named x-device-capability can be used to attach the class to a device capability.

Return type

swaggerFragment

Responses

200

The class was created or updated successfully.

400

The x-device-capability field refers to a capability that is not in the system.

500

server error

get /devices
List of devices. (deviceList)
Information about each device such as hardware capabilities (classes) and apps deployed. This list can be filtered based on device information or based on the apps running on the devices. The filter expressions are based on CSS selectors. You can for example filter based on device or app classes .canPlaySound or based on attributes [location=TE113]. You can also build more complecs expressions e.g. .canPlaySound[location=TE113].

Query parameters

device (optional)
Query Parameter — Filter based on device's information.
app (optional)
Query Parameter — Filter based on applications

Return type

array[device]

Responses

200

List of devices. If no filtering parameters all of them.

400

Error in app or device filter expression or with database.
